# TagsRow

A slim horizontal row rendered above the request URL bar. It displays the tags applied to the current request as coloured chips and provides controls to apply, create, recolour, and remove tags from the collection's shared tag registry.

Source: [`src/components/composer/TagsRow.tsx`](../../../components/composer/TagsRow.tsx)

> **Store-seeded stories.** TagsRow reads `requestStates[requestId].tags` from `requestStore` and `collections[collectionPath].manifest.tag_registry` from `collectionStore`. Every story pre-seeds both stores via `beforeEach` using the stable IDs `story-tagsrow-req-001` and `story-collection`.

## Behaviour Notes

- **Tag registry** â€” tags are defined once per collection in `manifest.tag_registry`. Applying a tag to a request stores only the tag name string in `requestStates[id].tags`; the color is resolved from the registry at render time.
- **Pending cache** â€” when a new tag is created, the definition is optimistically cached in local component state (`pendingTagDefs`) so the chip renders with the correct colour before the Tauri `save_tag_registry` round-trip completes.
- **Recolouring** â€” clicking a chip's coloured dot opens `ColorSwatchPopover`. Selecting a swatch calls `updateTagRegistry` to persist the new color for all requests that share this tag.
- **Empty state** â€” when no tags are applied and the popover is closed, only the "+ Add tag" label is rendered. When at least one tag is applied a bare "ďĽ‹" button appears at the end of the chip list.
