# MethodSelector

The HTTP method / protocol dropdown pill displayed at the left edge of the request URL bar. Clicking the pill opens a layered dropdown with HTTP methods, protocol types, and a free-text custom method entry. Each standard method maps to a dedicated design-token colour pair so the pill is colour-coded at a glance across all 13 Cortex themes.

## Custom Method

Non-standard methods (e.g. `PURGE`, `LOCK`, `MKCOL`) are entered via the freeform text field at the bottom of the dropdown. They render in a neutral grey pill because no dedicated design token exists for them. Pressing **Enter** or clicking **Apply** commits the value and closes the dropdown.

## Props

| Prop       | Type                       | Description                                                                                                                                             |
| ---------- | -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `method`   | `string`                   | The currently selected HTTP method or protocol. Displayed in uppercase; matched case-insensitively against the known method list for colour resolution. |
| `onChange` | `(method: string) => void` | Callback fired when the user selects a method from the dropdown or submits a custom method.                                                             |

## Behaviour Notes

- **Click outside to close** — a `mousedown` listener on `document` closes the dropdown when the user clicks outside the component.
- **Custom method via keyboard** — pressing **Enter** in the custom method `<input>` submits without requiring a click on **Apply**.
- **Tooltip alignment** — the trigger button tooltip is `align="start"` so it does not overlap the dropdown.
- **Pill width** — fixed at `72px` (`w-[72px]`) to prevent layout shift when switching between short (`GET`) and longer (`DELETE`, `GraphQL`) method labels.
