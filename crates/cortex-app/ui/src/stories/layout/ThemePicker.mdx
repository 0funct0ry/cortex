# ThemePicker

A compact overlay that lists all 13 Cortex themes grouped into **Light Themes** and **Dark Themes** sections. The currently active theme is indicated by a filled accent dot and a "✓ active" label. Hovering a theme previews it instantly by updating `document.documentElement.dataset.theme`; moving the cursor away reverts to the active theme. Clicking a theme commits the selection and closes the picker. Pressing Escape or clicking outside the overlay also closes it.

## Hover preview

`onMouseEnter` on each theme row calls `document.documentElement.dataset.theme = themeId`, which instantly swaps all CSS custom-property tokens without persisting the change. `onMouseLeave` reverts to the stored active theme. The cleanup function in the `useEffect` also reverts on unmount so closing the picker without selecting always restores the previous theme.

## Persistence

`setTheme(themeId)` (from `ThemeContext`) writes to `localStorage` under the key `cortex.theme`. On next load `ThemeProvider` reads this key and initialises the theme before the first render, preventing a flash of the default theme.

## Context dependency

`ThemePicker` calls `useTheme()` and will throw if rendered outside a `ThemeProvider`. Stories wrap the component with `ThemeProvider` in a decorator and pre-seed `localStorage` via `beforeEach` to control the initial active theme.
