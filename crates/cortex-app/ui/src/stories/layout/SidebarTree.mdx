# SidebarTree

The collapsible tree panel on the left edge of the workspace. It renders the full collection hierarchy — collections, folders, and individual requests — and is the primary navigation surface for saved API collections. Each node is drag-and-drop reorderable, context-menu enabled, and opens a tab in the composer when clicked.

## Store dependencies

The `SidebarTree` component reads from multiple Zustand stores. In Storybook, these are seeded via `beforeEach` after the global store-reset decorator runs:

| Store                | Seeded fields                            |
| -------------------- | ---------------------------------------- |
| `useWorkspaceStore`  | `activeWorkspace`, `activeWorkspacePath` |
| `useCollectionStore` | `collections` map, `expansionState` map  |

The `TabsProvider` context is required because clicking a request node calls `openTab()` from `useTabs()`.

## Tauri IPC mocking

The global IPC mock (applied by the `withTauriMock` decorator) nullifies all `invoke()` calls by default. Individual stories can override specific commands via `parameters.tauriMock`:

```tsx
parameters: {
  tauriMock: {
    my_command: () => ({ some: 'data' }),
  },
}
```

**Important**: mock return values must be the raw inner data — the binding wrapper automatically wraps them in `{ status: "ok", data: … }`. Returning a pre-wrapped object will result in double-wrapping.

## Collection node structure

Each collection in the sidebar maps to a `.crx` YAML file on disk. The tree structure mirrors the YAML hierarchy:

```
Collection (root)
├── Folder
│   ├── Request
│   └── Request
└── Request
```

Requests are colour-coded by HTTP method using the same `text-method-*` tokens as `TabItem`. Folder nodes show expand/collapse chevrons and a folder icon.
