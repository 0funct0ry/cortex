# Token Gallery

Cortex's visual language is built on **39 CSS custom properties** (design tokens) defined in `src/styles/tokens.css`. Every component uses tokens — never hardcoded colours — so themes only need to override these properties under their own `[data-theme="<id>"]` selector.

This gallery renders all 39 tokens as named swatches, grouped by category. Token values are read live from the computed document styles, so switching the **Theme** toolbar at the top of the page updates every swatch automatically.

## Token categories

### Background (6)

Background tokens define the layered surface system. Each layer sits on top of the previous one, creating depth without shadows.

| Token                  | Role                                              |
| ---------------------- | ------------------------------------------------- |
| `--color-bg-base`      | Page / window root background — the darkest layer |
| `--color-bg-panel`     | Sidebar, panel chrome — one step above base       |
| `--color-bg-surface`   | Cards, list items, editor background              |
| `--color-bg-overlay`   | Dropdowns, popovers, context menus                |
| `--color-bg-muted`     | Subtle fills for inactive or disabled areas       |
| `--color-bg-highlight` | Selection highlight, hover state fill             |

### Text (5)

| Token                    | Role                                    |
| ------------------------ | --------------------------------------- |
| `--color-text-primary`   | Default body text, headings             |
| `--color-text-secondary` | Supporting labels, metadata             |
| `--color-text-muted`     | Placeholders, disabled text, timestamps |
| `--color-text-inverse`   | Text on accent/filled backgrounds       |
| `--color-text-link`      | Hyperlinks, interactive text            |

### Border (3)

| Token                    | Role                                  |
| ------------------------ | ------------------------------------- |
| `--color-border-subtle`  | Dividers, hairlines — lowest contrast |
| `--color-border-default` | Input outlines, card borders          |
| `--color-border-strong`  | Focused input, selected item outline  |

### Accent (3)

The brand accent is the single colour used for primary actions (buttons, active state indicators, links on dark surfaces).

| Token                       | Role                                              |
| --------------------------- | ------------------------------------------------- |
| `--color-accent`            | Primary brand colour                              |
| `--color-accent-hover`      | Darkened accent for hover / active states         |
| `--color-accent-foreground` | Text / icon colour on an accent-filled background |

### Status (8)

## How tokens are defined

All 39 tokens have fallback values on `:root` in `src/styles/tokens.css`. Each theme overrides them under its own attribute selector:

```css
/* src/styles/tokens.css — fallback defaults */
:root {
  --color-bg-base: #000000;
  --color-accent: #000000;
  /* ... all 39 tokens ... */
}

/* src/styles/themes/dark.css — theme override */
[data-theme='dark'] {
  --color-bg-base: #1a1a1a;
  --color-accent: #f6ad55;
  /* ... all 39 tokens ... */
}
```

Because the selector is element-agnostic (`[data-theme]`, not `:root`), any DOM element can act as a theme root — this is how `ThemeGallery` renders all 13 themes simultaneously without global conflicts.

## Using tokens in components

Always reference tokens via CSS custom properties or Tailwind utility classes, never with hardcoded values:

```tsx
/* ✅ Correct — responds to theme changes */
<div className="bg-bg-panel text-text-primary border border-border-default rounded-md">
  <span className="text-text-muted">Metadata</span>
</div>

/* ✅ Also correct — inline CSS var */
<div style={{ color: 'var(--color-accent)' }}>Accent text</div>

/* ❌ Wrong — hardcoded colour breaks on theme switch */
<div style={{ color: '#f6ad55' }}>Accent text</div>
```

The Tailwind config in `tailwind.config.ts` maps every token to a utility class:

| CSS variable             | Tailwind class                              |
| ------------------------ | ------------------------------------------- |
| `--color-bg-base`        | `bg-bg-base`                                |
| `--color-text-primary`   | `text-text-primary`                         |
| `--color-border-default` | `border-border-default`                     |
| `--color-accent`         | `bg-accent`, `text-accent`, `border-accent` |
| `--color-method-get`     | `text-method-get`, `bg-method-get`          |
| `--radius-md`            | `rounded-md`                                |

## Adding a new token

1. Add the token with a fallback value in `src/styles/tokens.css` under `:root`.
2. Add a themed value in every theme file under `src/styles/themes/`.
3. Add the Tailwind mapping in `tailwind.config.ts` if the token needs a utility class.
4. Add the token to the `CATEGORIES` array in `TokenGallery.stories.tsx` so it appears in this gallery.

A token that appears as solid black (`#000000`) in the gallery indicates step 2 was missed for the active theme — the fallback from `tokens.css` is showing.
